## 1. なぜ「月10万円」なのか

月10万円は、個人開発者にとって最初の意味ある閾値です。

- 副業として成立する金額（年間120万円）
- サーバー代・ツール代を差し引いても利益が出る
- 「趣味」から「事業」に変わるライン
- 確定申告が必要になる実感のある金額

月100万円を最初から狙うのは非現実的です。まず10万円の仕組みを作り、それをスケールする方が確実です。

## 2. 前提条件と必要なスキル

このロードマップは以下の前提で書いています：

- **プログラミング経験**: 1年以上（言語は問わない）
- **使用するAIツール**: Claude Code（メイン）+ ChatGPT/Gemini（補助）
- **週の作業時間**: 10-15時間
- **初期投資**: 月額約¥5,000（Claude Pro + ドメイン代）

特別なスキルは不要です。「コードが書ける」「日本語で文章が書ける」——この2つがあれば十分です。

## 4. Phase 2（3-4週目）：最初の¥13,000

Phase 1で基盤ができたら、いよいよ収益化を始めます。


無料記事で読者を集め、深掘り記事を有料にします。

**価格設定の鉄則:**
- ¥300-500: Tips系（コマンド集、チートシート）
- ¥1,000-1,500: 実践ガイド（ステップバイステップ）
- ¥3,000+: 体系的な教材（コース的な深さ）

**成功パターン**: 無料記事で「何が」できるかを見せ、有料記事で「どうやって」を教える。

```
例:
無料: 「Claude Codeで開発速度が10倍になった方法」（What）
有料: 「CLAUDE.md完全攻略テンプレート集」（How）
```

### 収益源②：アフィリエイト — 目標: ¥3,000-5,000

技術記事にアフィリエイトリンクを自然に組み込みます。

おすすめASP:
- **A8.net**: クラウドサービス案件（DigitalOcean, AWS等）
- **楽天アフィリエイト**: 技術書・ガジェット紹介
- **もしもアフィリエイト**: Amazon経由の書籍紹介

**コツ**: 記事の中で「実際に使っている」ツールを紹介する形にする。広告感を出さない。

### 収益源③：クロスパブリッシュ — 目標: ¥3,000-5,000

1つの記事を複数プラットフォームに展開します。

```
  ├→ Zenn（技術特化版・バッジ収益）
  ├→ Qiita（コミュニティ版・LGTM獲得）
  ├→ dev.to（英語版・グローバルリーチ）
  └→ X/Twitter（要約スレッド・トラフィック誘導）
```

1記事を書く労力で4プラットフォームに展開。各プラットフォームの読者層が異なるため、カニバリゼーションは最小限です。

**自動化のポイント**: Claude Codeに「この記事をZenn形式に変換して」と指示すれば、フロントマター・タグ・文体を自動調整してくれます。

## 5. Phase 3（5-8週目）：¥30,000-50,000へスケール

Phase 2で月¥13,000の基盤ができたら、コンテンツ量を増やしてスケールします。

### 週次コンテンツルーティン

| 曜日 | タスク | 時間 |
|------|--------|------|
| 月 | 新規記事の執筆（Claude Code活用） | 2h |
| 水 | クロスパブリッシュ＋SNS投稿 | 1h |
| 金 | 分析＋翌週のトピック選定 | 1h |
| 土 | 有料コンテンツ作成 or メンテ | 2h |

週6時間で回るルーティンです。

### ニュースレターの追加 — 月¥5,000-10,000

Phase 3でニュースレター（Beehiiv等）を追加します。

- 無料読者を集める（記事CTAで誘導）
- 有料プラン（月額¥500）でプレミアムコンテンツ提供
- 20人の有料読者で月¥10,000

### ストック型コンテンツの強化

記事は「フロー型」と「ストック型」があります。

- **フロー型**: 最新ニュース、トレンド解説 → 短期的なアクセス
- **ストック型**: チュートリアル、テンプレート集、ベストプラクティス → 長期的に検索流入

Phase 3ではストック型の比率を70%以上にします。1年後も検索される記事が、安定収益の基盤になります。

記事で専門性を証明し、サービスで単価を上げる。この組み合わせが月10万円の壁を超えるポイントです。

## 7. 収益チャネル別の戦略

- 無料:有料 = 2:1の比率
- 有料記事は「シリーズもの」にする（Part 1無料 → Part 2, 3有料）
- マガジン機能でまとめ買い促進

### Zenn戦略
- 技術深度を最優先（Qiitaより専門的に）
- Bookを活用（体系的な内容 → バッジ収益）
- スクラップで日常的な学びを発信

### dev.to戦略
- 英語圏の読者に向けて日本のAI活用事例を紹介
- アフィリエイトCTAを記事末尾に配置
- シリーズ機能でリテンション向上

## 8. 自動化パイプラインの設計

手作業を減らすことが、少ない時間で月10万円を達成する鍵です。

### 自動化できるタスク

```
1. 記事テンプレート生成 → Claude Codeで下書き作成
2. クロスパブリッシュ → スクリプトで4プラットフォームに同時投稿
3. SNS投稿 → 記事公開時に自動ツイート
4. 分析レポート → 各プラットフォームのメトリクスを自動収集
5. トピック選定 → トレンド分析で最適なテーマを提案
```

### 最小限の自動化スタック

```bash
# cronで毎朝実行
0 9 * * * /path/to/daily-content-pipeline.sh

# パイプラインの中身:
# 1. 昨日の記事パフォーマンスを収集
# 2. 未配信の記事をクロスパブリッシュ
# 3. レポートを生成
```

重要なのは「全自動」ではなく「半自動」にすること。記事の品質チェックと最終公開判断は人間が行います。AIに丸投げした低品質記事は、ブランドを傷つけ長期的に損失になります。

## 9. よくある失敗パターンと対策

### ❌ 失敗①：最初から完璧を目指す
対策: Phase 1は「出す」ことが目的。品質は改善しながら上げる。

### ❌ 失敗②：1つのプラットフォームに依存
対策: 最低3プラットフォームに展開。アルゴリズム変更のリスク分散。

### ❌ 失敗③：トレンド記事だけ書く
対策: ストック型70%以上。「2026年版○○」は来年価値がなくなる。

### ❌ 失敗④：無料記事ばかり書く
対策: 無料:有料 = 2:1を守る。無料はファネルの入口。

### ❌ 失敗⑤：分析しない
対策: 週1回、各プラットフォームの数字を確認。伸びた記事のパターンを把握する。
